# BiLLiE website

A single-page marketing site with a qualification form. Practices find it, read
what BiLLiE does, submit their details, and Isaiah reaches out manually.

There is no product signup and **no database**. On submit, the form emails the
lead to `info@getbillie.com.au` (and sends the lead a courtesy auto-reply).

## This site is disposable

It has a planned death date: it gets rebuilt in **Framer** later. It is a
brochure, not the app. Keep it that way.

- **Zero coupling** to the BiLLiE monorepo. Do not import from, or change,
  anything in the app repo. Brand tokens are hardcoded in `tailwind.config.ts`.
- No auth, no sessions, no CMS, no shared packages, no persistence.

## Stack

- Next.js 15 (App Router), TypeScript strict
- Tailwind v3
- One route handler: `POST /api/lead` (Edge runtime, `syd1` / Sydney)
- [Resend](https://resend.com) for outbound email
- `@vercel/analytics` (cookieless)

## Structure

```
src/
  app/
    layout.tsx           fonts (Jost, Kalam), metadata, analytics
    page.tsx             composes the 9 sections
    globals.css
    api/lead/route.ts    zod validate -> Resend x2, honeypot, input caps
  components/            one component per section
  lib/
    copy.ts              SINGLE source of truth for every string. No copy
                         literals live in components.
    lead-schema.ts       one zod schema, shared by client + server
```

## Copy

All user-facing copy lives in `src/lib/copy.ts`. Components import from it.
Do not inline strings in components. The wordmark is exactly **BiLLiE**
(capital B, lowercase i, capital L, capital L, lowercase i, capital E).

## Environment

Copy `.env.example` to `.env.local` and set the key:

```
RESEND_API_KEY=re_...   # a key scoped to THIS project, not the app's auth key
```

Add the same variable to the Vercel project's environment. The Resend domain
`getbillie.com.au` is already verified (domain-wide), so `hello@` sends with no
new DNS. `hello@` is send-only and is **not** an inbox — auto-reply `replyTo`
is always `info@`.

## Develop

```
pnpm install
pnpm dev          # http://localhost:3000
pnpm typecheck
pnpm build
```

## Deploy

Push to `main` on `github.com/Isaiahjdehoog/billie-website`. Vercel builds and
deploys via the existing GitHub connection to **billie-website.vercel.app**.
The custom domain (`getbillie.com.au` / `www`) is **not** attached here — Isaiah
does that cutover manually.

## Brand assets

Favicon and OG image in `public/` are generated by `scripts/gen-assets.mjs`
(see the header comment in that file). The generator tooling is not needed to
build or deploy — the assets are committed.
